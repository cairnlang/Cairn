# Runewarden Project Vision

## Core Concept

`Runewarden` is a single evolving Cairn application that teaches the language by building something useful:
- academy operations for rune rituals,
- mine incident and inspection tracking,
- guild-role approval policies,
- web + persistence + actor workflows.

## Setting

The Stone Academy of Ironhold:
- Dwarven city setting with "stone/cairn" thematic fit.
- Strong fantasy flavor (runes, priests, guilds).
- Practical systems flavor (safety, logistics, accountability).

## Why This Fits Cairn

- Rich typed domain modeling.
- Clear pure-core/effect-shell boundaries.
- Natural use of `TEST`, `VERIFY`, and `PROVE`.
- Realistic path to web/auth/persistence/concurrency examples.

## Teaching Style

- Incremental milestones, one project throughout the full book.
- Every chapter ends with a runnable artifact.
- Frequent assurance checkpoints:
  - concrete `TEST`,
  - property `VERIFY`,
  - bounded proof `PROVE` where supported.

## Non-Goals (for first edition)

- No full framework abstractions.
- No "all features at once" big bang chapter.
- No hiding Cairn stack semantics behind too much magic.
